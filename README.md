- src/ — исходные файлы проекта

# Базовые классы

1.	Класс Api
(взаимодействие с сервером)

Свойства:

•	baseUrl: string – базовая часть url-адреса в виде строки
•	options: RequestInit - опции, которые можно передать функции fetch при совершении запросов на сервер. Свойство позволяет настроить различные аспекты запроса в данном случае заголовки

Конструктор

Конструктор класса принимает в качестве аргументов базовый Url для API и объект опций для запросов

Методы:

•	handleResponse(response: Response): Promise<object> - защищенный метод  - асинхронный обработчик ответа от сервера. Возвращает Promise с данными ответа или ошибкой
•	get(uri: string) - публичный метод принимает изменяемую часть url-адреса в виде строки, возвращает Promise с данными ответа в формате JSON, если запрос был успешным
•	Обрабатывает ошибки, возвращая сообщение об ошибке или текст статуса
•	post(uri: string, data: object, method: ApiPostMethods = 'POST') – публичный метод
- принимает uri - изменяемая часть url-адреса в виде строки, 
- принимает data - данные в виде объекта для отправки на сервер, 
- принимает method - метод  одного из вариантов типа ApiPostMethods = 'POST' | 'PUT' | 'DELETE'.
- 
Типы и интерфейсы:

•	type ApiListResponse<Type> - типизация ответа от сервера. Содержит общее число элементов и массив этих элементов заданного типа, полученных от сервера
•	type ApiPostMethods - типизация методов POST запросов на сервер

2.	Класс EventEmitter имлементирован IEvents
   (брокер событий)
  	
Класс EventEmitter реализуется с помощью паттерна Observer, что позволяет создать зависимость между объектами-наблюдателями и одним объектом-источником. При изменении состояния источника все наблюдатели автоматически об этом оповещаются. 
Данный класс выполняет роль контроллера – обеспечивает связь между моделью и отображением. EventEmitter обеспечивает работу с событиями – методы позволяют устанавливать и снимать слушатели событий, вызывать слушатель при возникновении события.

Свойства:

•	_events: Map<EventName, Set<Subscriber>> - приватное свойство, содержащее карту, где ключом является имя события (строка или регулярное выражение), а значением — набор подписчиков (функций-обработчиков)

Конструктор:

В конструкторе инициализируется свойство _events , которое использует Map  для хранения информации о событиях и их подписчиках.

Методы:

•	on<T extends object>(eventName: EventName, callback: (event: T) => void) - публичный метод принимает название события и коллбэк, обеспечивает подписку на событие (установку обработчика события)
•	off(eventName: EventName, callback: Subscriber) - публичный метод принимает название события и коллбэк, обеспечивает отписку от события (снятие обработчика события)
•	emit<T extends object>(eventName: string, data?: T) - публичный метод принимает название события и опционально данные.  Обеспечивает уведомление подписчиков о наступлении события.
•	onAll(callback: (event: EmitterEvent) => void) - публичный метод принимает коллбэк, позволяет подписаться на все события
•	offAll() - публичный метод позволяет отписаться от всех событий
•	trigger<T extends object>(eventName: string, context?: Partial<T>) - публичный метод принимает название события и опционально контекст, позволяет создать триггер и инициировать событие

Типы и интерфейсы:

•	type EventName - типизация имени события строкой или регулярным выражением
•	type Subscriber - типизация функции  установки подписки на событие
•	type EmitterEvent - типизация события эмиттера с именем и данными
•	interface IEvents – интерфейс, определяющий список методов и функций, которые планируются в любом классе, реализующем интерфейс. Нужен для реализации принципа инверсии зависимостей.
